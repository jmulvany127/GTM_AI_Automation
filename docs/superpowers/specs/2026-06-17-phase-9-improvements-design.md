# Design Spec — Phase 9 Improvements: Call Notes UI and Lead Page Enhancements

## 1. Overview

This spec covers every change needed for Phase 9. The goal is to make the call notes feature genuinely useful by adding context (title/description) to each analysis, building a properly filterable list dashboard, enabling inline call note submission from lead pages, improving the call analysis detail view, and adding quality-of-life improvements to the leads list and lead detail pages.

---

## 2. Database Changes

### 2.1 `call_analysis` table — two new columns

| Column | Type | Nullable | Notes |
|--------|------|----------|-------|
| `title` | VARCHAR(255) | YES | Human-readable call name, auto-generated by Claude if not supplied |
| `description` | TEXT | YES | 1–2 sentence summary of call context, auto-generated by Claude if not supplied |

**Migration file:** `alembic/versions/<hash>_add_title_description_to_call_analysis.py`

No other table changes are required.

## 6. UI Design Details

### 6.1 Call Notes List — Filter Bar Layout
```
[ Search title/lead... ] [ Lead ▾ ] [ Intent ▾ ] [ Linked ▾ ] [ From date ] [ To date ] [ Clear Filters ]
                                                                                    [ + New Call Analysis ]
```
All filters work together via JS. No page reload. Each filter reads `data-*` attributes on `<tr>` rows or hidden JSON.

### 6.2 Intent Score Badge Colours
- `score >= 7`: `.intent-badge-high` — green (#276749 on #c6f6d5)
- `4 <= score < 7`: `.intent-badge-mid` — amber (#92400e on #fef3c7)
- `score < 4`: `.intent-badge-low` — red (#c53030 on #fed7d7)
- No score: plain dash

### 6.3 Call Analysis Detail — Section Order
1. **Header card**: title (h2), description (p), lead link or "No lead linked", date badge, intent badge
2. **Call Intelligence Summary**: decision_timeline, recommended_follow_up, competitors
3. **Pain Points & Signals**: pain_points, budget_signals
4. **Objections**: objections
5. **Follow-Up Email**: pre-formatted block + Copy button
6. **CRM Note**: pre-formatted block + Copy button + HubSpot label if lead_id present

### 6.4 Lead Detail — Call Notes Section (Section 5)
```
Card header: "Call Notes"  [+ Add Call Note]
If empty:
  "No call notes yet."   [+ Add Call Note]

If has notes:
  compact table: Title | Intent | Date | [View]
  Below table: [+ Add Call Note]
```

### 6.5 Lead Detail — Run Agent Button
In the Lead Info card header:
- If `agent_run is None`: show `<button class="btn btn-primary" onclick="runAgent(...)">Run Agent</button>`
- If `agent_run` exists: show subtle `<p class="agent-last-run">Agent last run: 2025-06-01 14:23</p>`

The JS `runAgent()` function is identical to leads.html — POST to `/leads/{id}/run-agent`, disable button, reload on success.

### 6.6 New Call Analysis Form
- `title` — text input, optional, 255 char max
- `description` — textarea (3 rows), optional
- `lead_id` — `<select>` populated from `all_leads`, default option "No lead linked" (value `""`)
- `transcript` — `<textarea>` required, 10 rows
- Submit: `fetch('/call-notes/analyze', {method:'POST', body: JSON})`, loading state, redirect to `/dashboard/call-notes/{id}` on 200, inline error on failure

### 6.7 Leads List — Email Column
- Position: 3rd column, between Company and Title
- Render: `<a href="mailto:{{ item.lead.email }}">{{ item.lead.email }}</a>` or "—" if null
- `colspan` empty-state cell bumped from 8 to 9

## 7. AI Prompt Changes

Update `_SYSTEM_PROMPT` in `call_intelligence_service.py` to also return `title` (max 60 chars, e.g. "Discovery call with Sarah Chen — Greystar") and `description` (1–2 sentences summarising call context). The caller in `call_notes.py` should prefer values from the request body if supplied, falling back to AI-generated values.

---

## 8. Constraints

- No new external dependencies — stay within existing packages (FastAPI, SQLAlchemy, Jinja2, Anthropic)
- All filtering is client-side JS — no extra API endpoints for search
- Alembic migration must be generated and applied during Task 1
- Follow existing CSS naming conventions: `.badge-*`, `.btn-*`, `.card-*`
- Do not change unrelated files
- Conventional Commits for each task commit
